## 2. Call Path Diagrams (Observed)

### A. Reactive Description Generation
**Trigger**: Change in `brand`, `model`, `serial`, or `category`.
**Guard**: `shouldAutoGenerateDescription` returns `true` only if `!descriptionTouched` and `key !== lastAutoGenKey`.

1.  **Client**: `InventoryForm.tsx:144` (`useEffect`)
    *   Debounce: 800ms timer (`InventoryForm.tsx:147`)
    *   Calls `generateDescriptionSafely` (`InventoryForm.tsx:148`)
2.  **Client Logic**: `generateDescriptionSafely` (`InventoryForm.tsx:112`)
    *   Sets `isGeneratingDescription = true` (`InventoryForm.tsx:118`)
    *   Calls Server Action `generateDescriptionAction` (`InventoryForm.tsx:120`)
3.  **Server Action**: `lib/flood-engine/actions.ts:32`
    *   Dynamically imports service (`actions.ts:39`)
    *   Calls `generateApplianceDescription` (`actions.ts:40`)
4.  **Service**: `lib/flood-engine/services/geminiService.ts:431`
    *   **Attempt 1**: Call Gemini via `ai.models.generateContent` with `googleSearch` tool (`geminiService.ts:499`)
    *   **Fallback**: catch `searchError` -> Call Gemini WITHOUT tools (`geminiService.ts:524`)
5.  **Return**: JSON `{ description, seoKeywords }` (`geminiService.ts:550`)
6.  **Client Update**: `InventoryForm.tsx:130`
    *   Sets `description` state.
    *   Sets `seoKeywords` state.
    *   Updates `lastAutoGenKey` to prevent re-run.

### B. Specs Lookup (Auto-Fill)
**Trigger**: Button Click `handleSerialLookup` (`InventoryForm.tsx:283`)
**Purpose**: Populate original MSRP and age using deterministic serial decode first, then current/historical source evidence. The Wayback Machine check is part of the expected verification path for older model pages and MSRP evidence.

1.  **Client**: Calls `lookupApplianceSpecsAction` (`InventoryForm.tsx:288`)
2.  **Server Action**: `lib/flood-engine/actions.ts:23`
3.  **Service**: `lib/flood-engine/services/geminiService.ts:249`
    *   **Step 1**: Local Regex Decode `decodeAppliance` (`geminiService.ts:253`)
    *   **Step 2**: Gemini Search Query for MSRP (`geminiService.ts:307`)
    *   **Step 3**: Wayback Machine Check `checkWaybackAvailability` (`geminiService.ts:329`) for historical model/MSRP evidence when live pages are missing or stale.
4.  **Return**: JSON `{ msrp, ageMonths, notes, sources }`
5.  **Client Update**: `InventoryForm.tsx:290-292`
    *   Sets `originalPrice`, `ageMonths`, `serialLookupSources`. 
    *   **Chain**: Calls `generateDescriptionSafely` immediately (`InventoryForm.tsx:301`) to refresh description with new specs.

### C. Nameplate Extraction
**Trigger**: File Input Change `handleNameplateUpload` (`InventoryForm.tsx:255`)

1.  **Client**: `compressImage` -> Base64 (`InventoryForm.tsx:260`)
2.  **Client**: Calls `analyzeProductImageAction` (`InventoryForm.tsx:263`)
3.  **Server Action**: `lib/flood-engine/actions.ts:11`
4.  **Service**: `lib/flood-engine/services/geminiService.ts:94`
    *   Call Gemini Vision Model (`geminiService.ts:124`)
    *   Parse JSON response (`geminiService.ts:157`)
5.  **Return**: JSON Object
6.  **Client Update**: Mass state update (`InventoryForm.tsx:264-272`)
